# Roadmap Phases

## Phase 0 — Current state: validation MVP lab

Status: created.

Deliverables:

- local static MVP;
- feedback export;
- user-test script;
- call prep packet.

Decision gate:

```text
Do Tom and Ben agree that validation comes before full app spend?
```

## Phase 1 — Validation sprint

Duration: 1–2 weeks.

Deliverables:

- 5–10 user tests;
- exported feedback JSON per test;
- short findings memo;
- revised MVP scope;
- go/no-go recommendation.

Success signal:

- testers save movies;
- testers watch trailers;
- testers describe profile filter as useful;
- testers can explain the app in one sentence.

## Phase 2 — Data feasibility spike

Duration: 1–2 weeks.

Deliverables:

- metadata/trailer API proof;
- streaming availability API proof;
- theater/showtime feasibility proof for one market;
- provider abstraction plan;
- rough monthly API cost range;
- list of legal/terms constraints.

Decision gate:

```text
Can the app provide accurate enough routing at a cost that fits the business model?
```

## Phase 3 — Production MVP build

Duration: depends on vendor/team.

Recommended form:

```text
Responsive web/PWA first, with app-like UX, before native apps.
```

Core deliverables:

- production frontend;
- backend API;
- provider integrations;
- accounts/watchlist;
- analytics;
- admin curation tool;
- deployable hosting;
- privacy/terms basics.

## Phase 4 — Beta launch

Scope:

- limited user group;
- limited geography if theater routing exists;
- collect retention and outbound-click data;
- refine positioning.

## Phase 5 — Real launch or stop

Only move here if the beta shows retention, repeated use, and a believable path to revenue.
