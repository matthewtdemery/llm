# One-Page Call Brief

Call: Monday, June 29, 2026 at 3:00 PM EDT

## Goal of the call

Agree on the smallest credible next phase for MoviesMadeEasy / FindAMovie.

## Positioning

The idea is strongest when described as:

```text
A clutter-free, movies-only, trailer-first finder that uses a persistent taste profile — decade, classics, awards, genre, mood, and theater-vs-streaming preference — then routes the user toward where they can watch.
```

Avoid claiming:

```text
Nobody has combined movies, theaters, and streaming before.
```

That claim is too broad. The defensible wedge is the profile-filtered, movie-only, trailer-first experience.

## What we can show now

A validation-ready local MVP exists:

```text
[local path redacted]
```

It demonstrates:

- persistent profile filters;
- trailer-first movie cards;
- search;
- route filters for theater/streaming;
- mood/genre/decade/classic filters;
- save / trailer watched / not-for-me feedback;
- exportable feedback JSON;
- a user-test script.

## What it does not do yet

- No backend.
- No accounts.
- No real streaming availability.
- No real showtimes.
- No ticketing integrations.
- No paid APIs.
- No app store build.

That is intentional. This version tests whether the core flow is worth funding.

## Best next phase

```text
A 1–2 week validation sprint with 5–10 real users, using the MVP to measure whether people can find a movie they want in under 60 seconds.
```

## Proposed pass/fail signal

Pass if, out of 5 testers:

- 3+ save a movie;
- 3+ watch at least two trailers;
- 3+ say the profile filter is useful without being prompted;
- 2+ say they would use it instead of random scrolling for at least one use case.

## The key question for Tom and Ben

```text
Before we talk about building the full app, do we agree that the next phase should prove the profile-filter/trailer-first wedge with real users?
```
