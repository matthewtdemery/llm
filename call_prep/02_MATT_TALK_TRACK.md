# Matt Talk Track

## Opening — 30 seconds

```text
Tom, thank you again. I enjoyed talking with you too, and I think there is something real in the idea. I also want to protect you from spending full-app money too early. What we have now is enough to validate the strongest part of the idea before anyone commits to a big build.
```

## The honest but positive framing

```text
I do not think the opportunity is simply “the only app for movies, theaters, and streaming.” A lot of products touch those pieces already. The stronger idea is more specific: a movie-only, trailer-first finder with a persistent taste profile that cuts the clutter and helps someone decide what to watch faster.
```

## What changed since the last discussion

```text
I took the concept past a static mockup into a small MVP lab. It still uses demo data, but it now has profile persistence, search, mood and genre filters, route filters, save/not-for-me feedback, trailer tracking, and a feedback export. So it is not a production app, but it is enough to test whether people actually care about the core flow.
```

## The main recommendation

```text
My recommendation is not to start with a full native app quote. I would start with a validation sprint: show this to 5 to 10 people, watch whether they can find a movie in under a minute, capture their feedback, and then decide what the first real build should include.
```

## The vendor caution

```text
The UI is not the hard part. The expensive part is reliable data: movie metadata, trailers, streaming availability, theater showtimes, ticket links, deep links, and provider terms. So before choosing a developer, I would ask for a data-source plan, not just a design plan.
```

## Invite Tom/Ben into the conversation

```text
I want to hear both of your comments first. Tom, I especially want to know which parts of the concept matter most to you emotionally and practically. Ben, I want your skepticism too — where do you think this breaks, gets expensive, or becomes hard to market?
```

## Close / ask for decision

```text
If we agree, the next phase is simple: run a short MVP validation sprint, document results, then use those results to decide whether to build a production MVP and what data providers it actually needs.
```
