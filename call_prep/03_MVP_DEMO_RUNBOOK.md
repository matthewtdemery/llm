## Demo goal

Do not demo every feature. Demo the user story:

```text
I want to find a movie quickly without TV clutter, endless scrolling, or guessing where I can watch it.
```

## 5-minute demo sequence

### 1. Start with the hero

Say:

```text
This is intentionally labeled as demo data. The point is to validate the decision flow, not pretend we already have live licensing and routing solved.
```

### 2. Show profile filters

Set a profile like:

```text
Route: All
Mood: Classic or Award Winner
Genre: Drama or Crime
Decade: 1940s–2020s
Classic exceptions: on
```

Say:

```text
This is the wedge: the app remembers taste and cuts clutter before the user sees the grid.
```

### 3. Show trailer-first cards

Open one trailer card. Mention:

```text
The trailer is first because the user is trying to decide, not read a database entry.
```

### 4. Show save/not-for-me feedback

Click Save on one movie and Not for me on another.

Say:

```text
Even before accounts or analytics, this gives us a way to observe whether the concept is generating intent.
```

### 5. Show export feedback JSON

Click Export feedback JSON if appropriate.

Say:

```text
For the validation sprint, every test can produce a receipt. We are not guessing whether the test worked.
```

### 6. Show user-test script

Click User-test script.

Say:

```text
This is the next phase I recommend: small user test first, full app decision second.
```

## Demo phrases to avoid

Avoid:

```text
This already solves streaming and showtimes.
```

Use:

```text
This proves the UX and gives us a clean seam where live data would plug in later.
```
